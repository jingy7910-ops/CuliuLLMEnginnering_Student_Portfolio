# 阶段开发记录（约300字）

本周完成了 MVP 第一条链路的可运行原型：从 SGF 读取棋谱、解析指定手数，并生成基础复盘说明。`parse_sgf.py` 已支持输出对局双方、总手数、前10手、指定手的 SGF 坐标与棋盘语义区域（如左上角、中腹附近、边路三线附近）。

测试方面，使用了 3 份样本覆盖布局、中盘与官子阶段。布局样本中，前10手坐标与区域描述一致性良好；中盘样本可稳定解析连续战斗落点；官子样本可正确定位后半盘手数并输出对应描述。复盘文本均超过100字，结构满足“当前局面位置-可能意图-初学者建议”。

失败与问题记录：
1. 区域语义目前采用阈值规则，少数临界点可能出现“角/边/中腹”判定不够贴合人类棋感的问题。
2. 当前版本未接入真实模型推理，使用的是 Prompt 原型与本地草拟文本，尚未验证不同模型在术语稳定性上的差异。
3. 未处理复杂 SGF 分支变化图（仅按主线解析），后续需要加入变着分支支持与错误输入容错。

下一步将接入 Ollama/Qwen 做真实生成评测，记录坐标错误、术语误用和表达不清案例，再迭代 Prompt。
